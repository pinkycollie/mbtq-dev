## 2024-11-20 - [Fix SSRF vulnerability using custom agents in axios]
**Vulnerability:** A Server-Side Request Forgery (SSRF) risk existed because webhook URLs were not fully validated against pointing to local/private network ranges before the `axios.post` request.
**Learning:** Checking the IP resolution of a URL before handing it off to `axios` is not enough to stop SSRF because an attacker could use DNS rebinding (TOCTOU). `axios` performs its own lookups, which an attacker could exploit with dynamic DNS TTLs.
**Prevention:** To fully prevent SSRF and DNS Rebinding attacks in Node.js/axios, you must use a custom `http.Agent` and `https.Agent` with a custom `lookup` function that resolves the domain and validates the IP (using a library like `ipaddr.js` to block private/internal ranges) exactly at the socket connection level.
